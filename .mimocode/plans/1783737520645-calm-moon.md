# Plan: Fix Ofertas - Guardado, Preview y Pricing

## Problemas encontrados (auditoría)

1. **OfertasTab no guarda** — El auto-save en ProductFormContext no envía `variants`, `ctaText`, `crossSellProductIds`, `prices`, `enabledPriceTypes` al endpoint draft
2. **Campos de precio no aparecen** — OffersTab no muestra inputs de precio/precio tachado para cada oferta
3. **Preview no muestra checkout** — Solo muestra CTA animado, falta el modal de checkout simulado
4. **Resumen checkout sin descuento** — El precio del producto no aplica descuento de la oferta seleccionada
5. **Imports no usados** — ShoppingCart, Minus, Plus, useCartStore, calculatedDiscount
6. **Duplicate key** — `variants` se setea 2 veces en el producto

## Fixes

### Fix 1: ProductFormContext auto-save
Agregar al body de `triggerAutoSave`:
- `variants`, `prices`, `enabledPriceTypes`, `ctaText`, `crossSellProductIds`, `productImages`, `mainImageIndex`

**Archivo:** `wms/src/components/catalogo/ProductFormContext.tsx` (líneas ~659-682)

### Fix 2: OffersTab - campos de precio visibles + descuento manual
Cada oferta en OffersTab debe mostrar:
- Nombre de la oferta
- Imagen (1 sola)
- **Precio final (S/)** — input numérico
- **Precio original tachado (S/)** — input numérico (el admin pone este valor manualmente)
- **Badge** — dropdown
- **% descuento** — se calcula automáticamente del precio y tachado, solo se muestra (no editable)

El admin ingresa precio final y precio tachado. El % se calcula: `Math.round((1 - price / compareAtPrice) * 100)`.

**Archivo:** `wms/src/components/catalogo/tabs/OffersTab.tsx`

### Fix 3: WebPreview + MobilePreview — checkout simulado SIEMPRE visible
La preview del checkout debe mostrarse SIEMPRE, no solo en el tab de ofertas. El componente `CheckoutPreview` se renderiza al final de la preview, debajo de los bloques de landing.

Implementar `CheckoutPreview` inline que muestre:
- Tarjeta del producto con imagen, nombre, precio con descuento tachado, badge
- Lista de ofertas como radio cards (selección funcional)
- Formulario de envío EXACTO al checkout real: nombre, teléfono (9 dígitos), selects UBIGEO funcionales, dirección, referencia
- Productos sugeridos si existen
- Resumen con subtotal, envío, total
- Botón "COMPRAR AHORA" animado
- Pago: Contraentrega / MercadoPago
- NO envía pedidos (es solo visual)

**Descuento manual:** El admin pone precio final y precio tachado por separado. El % se calcula automáticamente para mostrar visualmente.

**Archivos:**
- `wms/src/components/catalogo/preview/WebPreview.tsx`
- `wms/src/components/catalogo/preview/MobilePreview.tsx`
- Crear `wms/src/components/catalogo/preview/CheckoutPreview.tsx` (nuevo componente compartido)

### Fix 4: CheckoutModal — precio con descuento en resumen
El resumen del checkout ya muestra el precio del offer. Verificar que:
- `offer.price` se usa correctamente (no `product.price`)
- El subtotal suma `offer.price + extraItems[].price`
- El descuento se muestra con precio tachado

**Archivo:** `tienda/src/components/checkout/CheckoutModal.tsx`

### Fix 5: Limpiar imports y duplicate key
- Remover imports no usados de WebPreview, MobilePreview, CheckoutModal, [slug]/page.tsx
- Eliminar duplicate `variants` key en [slug]/page.tsx
