# Plan: GEO (Generative Engine Optimization) + Fixes Pendientes

## Resumen
Implementar GEO completo para que las IAs (ChatGPT, Perplexity, Gemini) entiendan, indexen y recomienden la tienda. Además crear las 5 API routes faltantes que rompen las páginas del WMS.

---

## FASE 0: Fixes - API Routes Faltantes

Las páginas del WMS llaman endpoints que no existen. Crearlos:

### 0.1 Analytics Avanzado
- **Crear** `wms/src/app/api/v1/analytics/advanced/route.ts`
- GET con query param `type=clv|cohort|seasonal|cart`
- CLV: calcular total_gastado × frecuencia por cliente desde BD
- Cohort: agrupar clientes por mes de registro, calcular retención
- Seasonal: ventas agrupadas por mes últimos 12 meses
- Cart: tasa conversión, valor promedio, productos abandonados

### 0.2 Reportes IGV
- **Crear** `wms/src/app/api/v1/reports/igv/route.ts`
- IGV mes actual, IGV año, base imponible, desglose mensual
- Consultar invoices con taxAmount

### 0.3 Reportes Proveedores
- **Crear** `wms/src/app/api/v1/reports/suppliers/route.ts`
- Compras por proveedor, ranking, días promedio entrega
- Consultar purchaseOrders agrupadas por supplier

### 0.4 Reportes Márgenes
- **Crear** `wms/src/app/api/v1/reports/margins/route.ts`
- Precio venta vs costo, margen bruto %, top 10 rentables
- Calcular desde productVariants (price vs cost)

### 0.5 Reportes Inventario
- **Crear** `wms/src/app/api/v1/reports/inventory/route.ts`
- Valorización stock, rotación, obsoletos (>90 días sin movimiento)
- Stock por categoría, total unidades

### 0.6 Notificaciones API
- **Verificar** que `/api/v1/notifications/route.ts` exista
- Si no existe, crearlo (GET lista, POST crear)

### 0.7 Labels API
- **Verificar** que `/api/v1/labels/route.ts` exista
- Si no existe, crearlo (POST generar ZPL)

## FASE 1: GEO - Schema.org Completo

### 1.1 Product Schema (JSON-LD)
- **Modificar** `tienda/src/app/(public)/producto/[slug]/page.tsx`
- Agregar `<script type="application/ld+json">` con:
  - `@type: Product`
  - name, description, image, sku, brand
  - `offers`: price, priceCurrency (PEN), availability, url
  - `aggregateRating`: ratingValue, reviewCount (cuando haya reseñas)
  - `review`: author, reviewBody, ratingValue

### 1.2 FAQ Schema
- **Crear** `tienda/src/app/(public)/faq/page.tsx` — página visual FAQ
- **Crear** componente `FAQSchema.tsx` que genera JSON-LD
- 30+ preguntas organizadas por categorías:
  - Envíos y entregas (8-10 preguntas)
  - Pagos y devoluciones (8-10 preguntas)
  - Garantía y soporte (6-8 preguntas)
  - Productos y calidad (6-8 preguntas)
- Cada pregunta con `@type: FAQPage` + `mainEntity` → `Question` + `acceptedAnswer`

### 1.3 HowTo Schema
- **Crear** componente `HowToSchema.tsx`
- Guías de uso por producto (en sección oculta con `display:none` pero legible para IAs):
  - "Cómo armar la Cuna Convertible 3 en 1"
  - "Cómo ajustar la Silla Alta Ajustable"
  - "Cómo plegar el Cochecito City Mini"
- Cada guía con `@type: HowTo`, steps, tools, estimatedCost

### 1.4 LocalBusiness Schema
- **Modificar** `tienda/src/app/layout.tsx`
- Agregar `@type: LocalBusiness` con:
  - name, image, address, geo (lat/lng de Lima)
  - telephone, openingHours, priceRange
  - areaServed: "Peru"
  - hasOfferCatalog

### 1.5 BreadcrumbList Schema
- **Crear** componente `Breadcrumbs.tsx` visible + schema
- Mostrar en: Home > Categoría > Producto
- JSON-LD con `@type: BreadcrumbList`
- Agregar a páginas de producto y categorías

### 1.6 Review/Rating Schema
- **Modificar** modelo Prisma `Review` existente
- Cuando haya reseñas, generar `aggregateRating` en Product schema
- Crear endpoint `/api/v1/reviews` para obtener reseñas

### 1.7 Offer Schema
- Incluir en Product schema:
  - `@type: Offer`
  - price, priceCurrency, availability, validFrom, validThrough
  - seller: `@type: Organization` (AdriSu Kids)

### 1.8 Event Schema (promociones)
- **Crear** `tienda/src/lib/geo/event-schema.ts`
- Para promociones de temporada: Navidad, Día de la Madre, Black Friday
- `@type: Event`, name, startDate, endDate, offers, location

### 1.9 Video Schema (preparado)
- **Crear** `tienda/src/lib/geo/video-schema.ts`
- Template listo para cuando se agreguen videos
- `@type: VideoObject`, name, description, thumbnailUrl, uploadDate, duration
